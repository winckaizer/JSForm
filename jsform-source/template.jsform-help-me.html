🚀 Guía de Ayuda de JSForm

¡Bienvenido a JSForm! Esta guía te ayudará a entender la filosofía y el funcionamiento de este framework inspirado en la simplicidad y productividad de Windows Forms.

1. La Filosofía: WinForms en la Web

JSForm replica la arquitectura "Code-Behind" de .NET. Cada "Formulario" (o vista) se compone de tres archivos:

  • .html (La Vista): HTML puro. Aquí defines la estructura visual de tu componente.
  • .designer.js (El Diseñador): ¡NO TOCAR! Este archivo es auto-generado. Mapea cada elemento con un id en tu HTML a una propiedad en la clase del controlador (this.miBoton).
  • .controller.js (El Controlador): Aquí vive tu lógica. Hereda del diseñador, por lo que tienes acceso directo a los elementos del DOM a través de this.
¡Magia Automática! Cuando creas una nueva carpeta en app/forms, el compilador de JSForm (npm run dev) genera estos tres archivos por ti. Si modificas el HTML y guardas, el .designer.js se actualiza solo.

2. Estructura del Proyecto

  • /app/forms/: Contiene todas tus vistas/formularios. Cada subcarpeta es un módulo.
  • /app/services/: Ideal para crear clases que encapsulen la lógica de negocio (ej. UserService.js).
  • /assets/: Para tus imágenes, CSS y librerías de terceros.
  • /core/: El corazón del framework. No modifiques estos archivos, ya que se actualizan con el comando jsform update.
  • /dist/: Carpeta generada al ejecutar npm run build. Contiene la versión optimizada y ofuscada de tu app para producción.

3. El Auto-Binding de Eventos

Olvídate de addEventListener. JSForm lo hace por ti. Si tienes un botón <button id="btnGuardar"></button>, simplemente crea un método en tu controlador con el nombre btnGuardar_click(event). ¡Se enlazarán automáticamente!

// En tu home.controller.js
// ...
btnGuardar_click(e) {
    console.log("¡Botón Guardar pulsado!");
    // 'this' funciona correctamente aquí.
}

4. Componentes del Núcleo

JSForm.Control

Para crear elementos del DOM dinámicamente desde el código.

import { Control } from '../../core/JSForm.Control.js';

// ... en un método de tu controlador ...
const miDiv = new Control('div', 'divDinamico', {
    className: 'alert alert-info',
    html: '<strong>¡Hola!</strong> Soy un div creado por código.'
});
miDiv.renderTo(this.miContenedor); // this.miContenedor es un div con id="miContenedor" en el HTML

JSForm.MessageBox

Muestra diálogos modales asíncronos. Usa async/await para esperar la respuesta del usuario.

import { MessageBox } from '../../core/JSForm.MessageBox.js';

// Dentro de un método de tu controlador (ej. un evento de click)
async btnBorrar_click() {
    const resultado = await MessageBox.show(
        'Confirmar Acción',
        '¿Está seguro de que desea eliminar esto?',
        MessageBox.Buttons.YesNo,
        MessageBox.Icon.Warning
    );
    
    if (resultado === 'yes') {
        console.log('El usuario confirmó el borrado.');
        // Aquí iría la lógica para borrar el elemento...
    } else {
        console.log('El usuario canceló la operación.');
    }
}

JSForm.DataGridView

Un adaptador para la potente librería DataTables.js. Carga los scripts necesarios bajo demanda.

  1. En tu HTML: Añade una tabla vacía: <table id="gridUsuarios"></table>.
  2. En main.css: Descomenta la línea @import del CSS de DataTables.
  3. En tu Controlador:
import { DataGridView } from '../../core/JSForm.DataGridView.js';

export class MiControlador extends MiDesigner {
    constructor() {
        super();
        this.grid = null;
        this.init();
    }

    async init() {
        const datos = [{ id: 1, nombre: 'Ana' }, { id: 2, nombre: 'Luis' }];
        
        this.grid = await DataGridView.create('gridUsuarios', {
            dataSource: datos,
            columns: [
                { data: 'id', title: 'ID' },
                { data: 'nombre', title: 'Nombre' }
            ]
        });
    }

    onDestroy() {
        // ¡Muy importante para liberar memoria!
        if (this.grid) {
            this.grid.destroy();
        }
    }
}

JSForm.HttpClient

Un cliente para realizar peticiones a tus APIs de forma centralizada y sencilla.

  1. Configura tu API: Abre jsform.config.js y ajusta las URLs en la sección api.
  2. En tu Controlador:
import { HttpClient } from '../../core/JSForm.HttpClient.js';
import { MessageBox } from '../../core/JSForm.MessageBox.js'; // Opcional, para mostrar errores

// Dentro de un método de tu controlador
cargarUsuarios() {
    HttpClient.get('/users', {
        // Opcional: para usar una API no-default
        // api: 'jsonPlaceholder', 
        
        beforeSend: () => {
            console.log('Cargando...');
            // this.btnCargar.disabled = true;
        },
        success: (data, status) => {
            console.log('Usuarios recibidos:', data);
            // this.grid.setData(data); // Ejemplo de uso con DataGridView
        },
        error: (err, status) => {
            MessageBox.show('Error', `No se pudieron cargar los usuarios: ${err.message}`);
        },
        complete: () => {
            console.log('Petición finalizada.');
            // this.btnCargar.disabled = false;
        }
    });
}

5. Ejemplo Completo: Un Formulario de Login

Este es un ejemplo de cómo se verían las piezas juntas para un formulario de login.

1. Crea la carpeta app/forms/Login

El compilador generará login.html, login.designer.js y login.controller.js.

2. Edita login.html

<div id="loginPanel">
    <h2>Iniciar Sesión</h2>
    <input type="text" id="txtUsuario" placeholder="Usuario">
    <input type="password" id="txtPassword" placeholder="Contraseña">
    <button id="btnLogin">Entrar</button>
    <div id="lblMensaje"></div>
</div>

Al guardar, login.designer.js se actualizará solo con this.txtUsuario, this.txtPassword, etc.

3. Edita login.controller.js

import { LoginDesigner } from './login.designer.js';
import { HttpClient } from '../../core/JSForm.HttpClient.js';
import { MessageBox } from '../../core/JSForm.MessageBox.js';

export class LoginController extends LoginDesigner {
    constructor(parameters) {
        super();
        this.autoBindEvents();
        this.init();
    }

    init() {
        this.txtUsuario.focus();
    }

    // Evento auto-enlazado para el botón con id="btnLogin"
    async btnLogin_click() {
        const user = this.txtUsuario.value;
        const pass = this.txtPassword.value;

        if (!user || !pass) {
            this.lblMensaje.innerText = "Usuario y contraseña son requeridos.";
            return;
        }
        
        HttpClient.post('/auth/login', { user, pass }, {
            beforeSend: () => {
                this.btnLogin.disabled = true;
                this.lblMensaje.innerText = "Validando...";
            },
            success: (response) => {
                // Guardar token, redirigir, etc.
                console.log("Login exitoso:", response.token);
                // Ejemplo de navegación a otra vista:
                // import { HomeController } from '../Home/home.controller.js';
                // Application.run('home', HomeController);
            },
            error: (err, status) => {
                MessageBox.show('Error', 'Credenciales incorrectas', MessageBox.Buttons.OK, MessageBox.Icon.Error);
            },
            complete: () => {
                this.btnLogin.disabled = false;
                this.lblMensaje.innerText = "";
            }
        });
    }
}

4. Lanza el formulario en app/program.js

// En app/program.js
import { Application } from '../core/JSForm.Core.js';
import { LoginController } from './forms/Login/login.controller.js';

class Program {
    static async main() {
        Application.init();
        
        // Lanzamos nuestro formulario de Login como vista inicial
        await Application.run('login', LoginController);
    }
}

Program.main();

6. Comandos del CLI

  • npm run dev: Inicia el entorno de desarrollo con recarga en caliente.
  • npm run build: Compila y ofusca tu aplicación para producción. El resultado queda en la carpeta /dist.
  • jsform update: Actualiza el núcleo de JSForm a la última versión sin tocar tu código.
